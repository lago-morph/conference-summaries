Model Context Protocol(MCP) Authorization for Enterprise Use Case With Keycloak - Yutaka Obuchi, Hitachi, Ltd.

In the MCP community, Enterprise-Managed Authorization for MCP is proposed as Specification Enhancement Proposal #646(SEP-646, https://github.com/modelcontextprotocol/modelcontextprotocol/pull/646), which was proposed by one of Auth 2.1 spec authors and has just been merged as MCP Authorization Extensions on Oct 31st.

This proposal is intended to provide a solution for the authorization problem of MCP in enterprise use case and make
MCP authorization enterprise-ready from both ux and security perspective.

The target problem is that users need to manually log in and consent to grant an Al agent access to each external resources one by one and it is hard to visualize those access enough for enterprise security.
It will be getting bigger with future explosion of Al agents.

The session will cover the following:
- the target problem and the solution of SEP-646
- Keycloak’s implementation for RFC7523, which is prerequisite spec for SEP-646
- implementation of SEP-646 using Keycloak

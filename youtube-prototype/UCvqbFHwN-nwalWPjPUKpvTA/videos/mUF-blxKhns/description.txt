Advancing Machine to Machine Workflows with the SPIFFE Identity Provider - Andrew Block, Red Hat & Maia Iyer, IBM Research

Most Keycloak users start by configuring realms, clients, and roles by hand—but that doesn’t scale. As organizations adopt multi-tenant and cloud-native applications, manual configuration becomes a bottleneck and a source of security drift.

This session shows an approach for making Keycloak configuration declarative and repeatable. We’ll explore how to represent Keycloak realms and clients as code, how to bootstrap client credentials automatically, and how to connect Keycloak with trusted workload identities using SPIFFE/SPIRE so you'll never worry about copying a client secret ever again.

You’ll see real-world patterns for automating Keycloak setup across environments, including how to reconcile configuration drift, enforce consistent RBAC policies, and safely manage client registration at scale. By the end, you’ll know how to make Keycloak management as easy—and as reliable—as any modern DevOps workflow.
